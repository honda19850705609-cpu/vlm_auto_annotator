# Day 1 — VLM Environment Setup & Single-Image Inference
 
## Goal
Get a vision-language model running on the AutoDL instance and produce a
minimal inference script that takes one image and returns a description.
 
## Environment
- GPU: RTX 4080 SUPER, 32 GB VRAM
- torch 2.8.0+cu128, transformers 5.9.0, qwen-vl-utils 0.0.14
- Model: Qwen2.5-VL-7B-Instruct (bf16), stored on the data disk
  (`/root/autodl-tmp/qwen2.5-vl-7b`)
## What I did
- Pointed the HuggingFace cache to the data disk to avoid filling the small
  system disk.
- Wrote `minimal_vlm.py`: a single-image inference script with command-line
  arguments (model / image / prompt / max-new-tokens) and built-in latency and
  VRAM statistics.
- Ran inference on an aerial night-time city image. The model correctly
  identified it as an aerial view of a dense urban area at night and listed
  buildings, roads, streetlights, trees, open spaces, and vehicle light trails.
## Bugs hit & fixes
- Shell commands (`cd`, `export`, `source`) were accidentally written into the
  `.py` file, causing `NameError`. Fix: keep shell commands in the terminal and
  Python code in the file; build the file through the editor, not heredoc paste.
- HuggingFace download failed with `401 Unauthorized` from `xethub.hf.co`.
  Fix: `export HF_HUB_DISABLE_XET=1` to disable the xet backend, then download
  through a mirror endpoint with the new `hf download` CLI.
- The instance hostname changed between sessions (new container), so the system
  disk lost its installed packages. The data disk kept the model, so only
  `pip install` had to be re-run. Lesson: system-disk environment is volatile
  across reset/shutdown; data-disk files persist. Save an image to keep the env.
## Key data (from this run)
- Model load time: 7.8 s
- Generation time: 6.8 s for 312 tokens
- Throughput: 45.9 tok/s
- Peak VRAM: 16.0 GB (half of the 32 GB available)
## Takeaway
The full VLM pipeline runs end to end: image -> vision encoder -> projection ->
language model -> text. 16 GB peak VRAM leaves comfortable headroom for batch
inference and later edge-deployment optimization. Next step (Day 2): force the
free-form output into a stable structured JSON format.
